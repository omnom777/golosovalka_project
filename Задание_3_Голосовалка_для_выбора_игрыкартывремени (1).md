# Проект 3: Бот или голосовалка для выбора игры/карты/времени

**Сценарий**  
Разработайте сервис голосования, который помогает группе (классу) выбрать, во что поиграть, на какой карте или в какое время провести мероприятие. Это может быть:
- Telegram-бот (используя `python-telegram-bot` или `aiogram`);
- простой веб-сервис с API и минимальным фронтендом (кнопки голосования).

Основная задача: пользователи могут предлагать варианты и голосовать за существующие. Результаты голосования должны сохраняться.

**Роли и распределение задач**

| Ученик A (Логика голосования и хранение) | Ученик B (Интерфейс взаимодействия) |
|------------------------------------------|-------------------------------------|
| Создать модели: `Poll` (опрос) и `Vote` (голос) | Реализовать Telegram-бота или веб-интерфейс |
| Реализовать функции: создание опроса, добавление вариантов, голосование, получение результатов | Подключить логику к интерфейсу: команды бота или эндпоинты для веба |
| Хранить данные в `polls.json` (список опросов с вариантами и голосами) | Обеспечить удобное взаимодействие: например, `/create`, `/vote`, `/results` |
| Следить, чтобы один пользователь не голосовал дважды в одном опросе | Отображать текущие результаты в реальном времени или по запросу |

**Технические детали и подсказки**

- Пример структуры `polls.json`:
```json
{
  "polls": [
    {
      "id": 1,
      "question": "Во что играем?",
      "options": ["CS2", "Dota 2", "Minecraft"],
      "votes": {"user1": 0, "user2": 1, "user3": 0}
    }
  ]
}
```

- votes – словарь, где ключ – идентификатор пользователя (например, username или Telegram ID), значение – индекс выбранного варианта.

- Для Telegram-бота: используйте библиотеку python-telegram-bot. Пример создания бота:
```python
from telegram.ext import Application, CommandHandler

async def start(update, context):
    await update.message.reply_text("Привет! Я бот для голосования.")

application = Application.builder().token("YOUR_TOKEN").build()
application.add_handler(CommandHandler("start", start))
application.run_polling()
```

- Для веб-версии: можно сделать простой HTML с формами и кнопками, отправляющими запросы к FastAPI.

- FastAPI может служить бэкендом и для бота (бот отправляет запросы к вашему API), либо вся логика встроена в бота (тогда бот напрямую работает с JSON).

**Тестирование**
- Создайте опрос с тремя вариантами.
- Проголосуйте от имени двух разных пользователей (можно эмулировать разными запросами).
- Убедитесь, что повторное голосование блокируется или обновляется.

**Критерии приёмки**
- Пользователь может создать опрос с несколькими вариантами.
- Можно проголосовать (учитывается один голос от одного идентификатора).
- Результаты голосования корректно подсчитываются и выводятся.
- Проект работает стабильно, данные сохраняются между перезапусками.
